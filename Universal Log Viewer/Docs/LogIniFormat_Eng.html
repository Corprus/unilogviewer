Log Type Format Description

Log type filename should have *.ltp extension. This is not absolute requirement (yet).
Supported log type ini file settings:
  1. [Log File] Section should be present and should contain these parameters:
    1. RootBlock - root block type name
    2. LogName - log title
    3. Author - log type ini file author
    4. Version - log type ini file version
  2. Other structures:
    1. Common variables (for all types)
      1. Name - Type name
      2. Title - title of this log element
      3. Type - log element type
      4. TitleType - the type of this element's hown title (in present, this is only working with String). Field can take values: (Title, Value, Source). Title (default) - log element's type title is shown, Value - value of one of the children, Source - source parsed string
      5. Style - link to style type used in this element
    2. Block
      1. StartCondition, EndCondition - conditions of ending and starting the block
      2. ChildBlockTypes - CSV field, describing children (in the type's tree) of block type
      3. ChildStringTypes - CSV field, describing String type children
    3. String
      1. Condition - string condition (which is used to determine if current string belongs to this type of strings)
      2. UseSeparator - bool (default 0), determining if separatos algorithm or auto parsing to values if used
      3. Separator - CSV, array of possible separator symbols
      4. ChildValueTypes - CSV, possible value types
    4. Value
      1. Condition - Condition, by which we can determine if this value type is placed in current string and if yes - where.
      2. IncludeConditions - bool, default 1, condition if border condtiions is included in final value
      3. ValueType - (not used by this moment)
    5. Condition
      1. StartsWith - start condition (which string should start the current value)
      2. EndsWith - end condition
      3. Contain - CSV, strings which should be contained in this value
      4. Exclude - CSV, strings which should NOT be contained in this value
    6. Style
      1. Visible - bool, default 1, determine if this element is visible in final tree
      2. Bold - bool, default 0, if thislines should be bold in final tree
      3. Color - color NAME of this element, default black
