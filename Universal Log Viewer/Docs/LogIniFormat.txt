Назввание имени файла с типом должно иметь расширение *.ltp. Данное требование необязательное (пока).
Информация по поддерживаемым настройкам ini файла лога:

1. Должна присутствовать секция [Log File] с такими полями:
 1.1 RootBlock  - имя рутового (основного блока)
 1.2 LogName - имя лога 
 1.3 Author - автор
 1.4 Version - версия формата

2. Структурные типы лога:
 2.1 Общие переменные для всех типов
  2.1.1 Name - название класса (внутреннее) 
  2.1.2 Title - название класса (внешнее)
  2.1.3 Type - тип класа, для каждого класса свой (переменная поля должна совпадать с именем класса, указанное в данном документе)
  2.1.4 TitleType - тип отображаемого заголовка (пока что работает только с CString). Поле  может принимать значения:
        (Title, Value, Source). Title (по умолчанию) - название класса строки, Value - значение одного из детей (у каждого класса по своему определяется), Source - исходная строка
  2.1.5 Style - ссылка на соотв. тип стиля, определяет стиль отображаемого элемента
 2.2 Block
  2.2.1 StartCondition, EndCondition - имена типов (типа Condition), определяющих начало и конец блока
  2.2.2 ChildBlockTypes - CSV поле, определяющее список типов блоков, которые могут содержаться внутри данного блока (ТОЛЬКО ПЕРВЫЙ УРОВЕНЬ ВЛОЖЕННОСТИ)
  2.2.3 ChildStringTypes - CSV поле, определющее список типов строк которые (помимо блоков) могут содержаться в данном блоке
 2.3 String
  2.3.1 Condition - условие, по которому можно определить принадлежность строки к тому или иному типу
  2.3.2 UseSeparator - булевое значение (1,0) определяющее будет ли использоваться разделитель при парсинге строк (т.е. будет ли строка перед определение где расположены конкретные значения в строке разделяться по разделителю)
  2.3.3 Separator - разделитель(массив CSV). Используется только первый символ каждого элемента массива (если указана строка)
  2.3.4 ChildValueTypes - список типов значений (CSV), используемых в данной строке. Порядкозависимый, то есть принадлежность с типу значений определется в порядке указания данных типов
 2.4 Value
  2.4.1 Condition - условие, по которому можно определить входит ли данное значение в исходную строку, и если да - в каком месте расположено
  2.4.2 IncludeConditions - булевое значение (1,0) определяющее входят ли граничные условия из Condition в само поле значения
  2.4.3 ValueType - (на данный моент не используется) тип выдаваемых значений в поле
 2.5 Condition
  2.5.1 StartsWith - строка, с которой должна начинаться проверяемая строка
  2.5.2 EndsWith - условие на конец строки
  2.5.3 Contain - CSV, строки которые должна содержать проверяемая
  2.5.4 Exclude - CSV, строки которые НЕ должна содержать проверяемая
 2.6 Style
  2.6.1 Visible - булевое значение (0,1). По умолчанию 1 (то же самое если что либо кроме 0). Определяет будут ли в дереве отображаться элементы данного типа
  2.6.2 Bold - булевое значение, определяюще будет ли поле выделено жирным шрифтом
  2.6.3 Color - текстовое название (!) цвета, определяюще цвет шрифта поля
